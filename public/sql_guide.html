What is SQL?

SQL, or Structured Query Language, is a standardized programming language used to manage and manipulate relational databases. It allows users to perform various operations, such as querying data, updating records, inserting new data, and deleting existing records. SQL provides a way to interact with the database structure and define data schemas, as well as enforce data integrity and security. Common SQL commands include SELECT, INSERT, UPDATE, DELETE, CREATE, and DROP.

  • Relationships: Enforces relationships through constraints
  • Normalization: Supports normalization to reduce redundancy
  • Examples: MySQL, PostgreSQL, Oracle, SQL Server

1. DDL (Data Definition Language)

DDL involves the creation and modification of database structures. It establishes the framework for how data is organized and accessed. This includes defining tables, their columns, data types, constraints, and relationships between tables.

  • CREATE: Creates a new database object (e.g., table, index)
  • ALTER: Modifies an existing database object
  • DROP: Deletes a database object
  • TRUNCATE: Removes all rows from a table while keeping the structure
  • RENAME: Changes the name of an existing database object

üìã Characteristics of DDL

  • Schema Definition: DDL commands define the structure of the database schema
  • Database State Changes: Changes are usually permanent and cannot be rolled back
  • Object Creation: Allows defining attributes and constraints of database objects
  • Access Control: Some DDL commands can manage permissions and access control

üí° Use Cases of DDL

  • Creating Tables: Set up initial database structure for new projects
  • Modifying Structures: Add new fields or constraints as requirements change
  • Managing Database Objects: Clean up unused tables or reorganize schema

2. DML (Data Manipulation Language)

DML is focused on the actual data within the database. It allows users to add new data, modify existing records, or remove data as needed. This category is essential for everyday operations within a database.

  • INSERT: Adds new records to a table
  • UPDATE: Modifies existing records in a table
  • DELETE: Removes records from a table
  • MERGE: Combines data from two tables, updating or inserting

üìã Characteristics of DML

  • Data Handling: Focuses on manipulation of data stored in tables
  • Non-Permanent Changes: Operations can be rolled back if part of a transaction
  • User Interaction: Allows direct interaction with data
  • SQL Statements: Includes SELECT, INSERT, UPDATE, and DELETE

üí° Use Cases for DML

  • User Management: Adding, updating, or removing user records
  • Inventory Management: Tracking stock levels and product information
  • Order Processing: Managing sales orders and their statuses
  • Data Analysis: Retrieving specific data sets for reporting
  • Auditing and Logging: Maintaining logs of data changes

3. DQL (Data Query Language)

DQL is specifically concerned with retrieving data from the database. It allows users to query data, apply filters, and organize the results according to specific criteria.

  • SELECT: Retrieves data from one or more tables
  • JOIN: Combines rows from two or more tables based on related columns

üìã Characteristics of DQL

  • Data Retrieval: Specifically focused on querying data
  • Flexible Querying: Allows complex queries with filtering, sorting, and grouping
  • Non-Destructive: Read-only operations that don't alter data
  • SQL Statements: Primary command is SELECT with various clauses

üí° Use Cases for DQL

  • Reporting: Generating reports for business metrics
  • Data Analysis: Analyzing trends and patterns
  • User Queries: Searching for specific information
  • Dashboards: Building real-time data visualizations
  • Data Export: Retrieving data for export to other systems

4. DCL (Data Control Language)

DCL focuses on security and access control within the database. It defines who can access what data and what actions they are permitted to perform.

  • GRANT: Gives specific privileges to users or roles
  • REVOKE: Removes specific privileges from users or roles

üìã Characteristics of DCL

  • Access Control: Defines who can access or modify data
  • User Privileges: Allows granting and revoking privileges
  • Security Management: Maintains data integrity and confidentiality
  • Transactional: Can be part of transaction management

üí° Use Cases for DCL

  • User Management: Granting different access levels based on roles
  • Data Security: Protecting sensitive information
  • Role-Based Access Control: Implementing predefined permissions
  • Auditing and Compliance: Maintaining regulatory compliance

5. TCL (Transaction Control Language)

TCL is concerned with managing transactions in the database. A transaction is a sequence of operations that are treated as a single unit of work.

  • COMMIT: Saves all changes made during the current transaction
  • ROLLBACK: Reverts changes made during the current transaction
  • SAVEPOINT: Sets a point within a transaction to roll back to
  • SET TRANSACTION: Configures properties of the current transaction

üìã Characteristics of TCL

  • Atomicity: Transaction treated as single unit of work
  • Consistency: Maintains database integrity
  • Isolation: Manages transaction isolation to prevent interference
  • Durability: Committed changes are permanent

üí° Use Cases for TCL

  • Banking Transactions: Ensuring complete money transfers
  • Order Processing: Managing atomic e-commerce operations
  • Batch Processing: Performing multiple operations as one unit
  • Data Migration: Ensuring data integrity during transfers

Slowly Changing Dimensions (SCD)

SCD techniques are used to handle how historical data is tracked and updated when the values of dimensional attributes change.

Type 1

Overwrite (No History)

Characteristics: No historical data is maintained. The old value is overwritten with the new value.

Use When: Historical accuracy is not important or only latest information is needed.

Example: Customer address change - old address is replaced with new one.

Type 2

Add New Row (Full History)

Characteristics: Maintains full history. New row added for changes with start/end dates.

Use When: Important to track changes over time for historical reporting.

Example: New row inserted with new address, old row preserved with valid-to date.

Type 3

Add New Column (Limited History)

Characteristics: Tracks limited history with separate columns for current and previous values.

Use When: Only need to track current and previous value (limited history).

Example: Current_Address and Previous_Address columns in table.

Type 4

Add History Table

Characteristics: Separate history table stores changes while main table has current values.

Use When: Need to maintain clean main table while preserving historical data.

Example: Customer table + Customer_History table for past records.

Type 6

Hybrid Approach

Characteristics: Combines Type 1, Type 2, and Type 3 for different attributes.

Use When: Multiple attributes need different handling strategies.

Example: Address uses Type 2 (full history), phone uses Type 1 (overwrite).

  • Type 1: When only current data matters (e.g., employee role)
  • Type 2: When full historical record is crucial (e.g., customer addresses)
  • Type 3: When limited history is important (e.g., current and previous address)
  • Type 4: When separating current and historical data optimizes performance
  • Type 6: When different attributes require different strategies
