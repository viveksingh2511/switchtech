Overview of Data Types

Python provides several built-in data types used to represent different kinds of values. Understanding data types is fundamental to writing efficient, bug-free code and is crucial for technical interviews.

Data types in Python are categorized into several groups based on their characteristics and use cases:

Category Data Types Description
Numerical int, float, complex Represent numeric values
Text str Represents text data
Sequence list, tuple, range Ordered collections of items
Mapping dict Key‚Äìvalue pairs
Set Types set, frozenset Unordered collection of unique items
Boolean bool Logical True or False values
NoneType None Represents the absence of value
Interview Tip

Key Points to Remember:

  • Python is dynamically typed ‚Äî you don't need to declare types explicitly
  • Use type() to check a variable's data type
  • Mutability is crucial: list, dict, set are mutable; tuple, str, int, float are immutable

Numerical Types

int float complex

1. Integers (int)

Integers represent whole numbers, positive or negative, without decimals. Python 3 integers have arbitrary precision, meaning they're not limited by 32 or 64 bits like in many other languages.

PYTHON 
a = 1000000000000000
print(a)                    # 1000000000000000
print(type(a))              # <class 'int'>
print(a.bit_length())       # Bits used to store the number

# Negative integers
b = -42
print(b, type(b))           # -42 <class 'int'>

# Integer operations
x = 10
y = 3
print(x + y)                # 13
print(x * y)                # 30
print(x ** y)               # 1000 (exponentiation)
Interview Tip

Python's ability to handle large integers is leveraged in cryptography, finance, and machine learning (e.g., computing factorials or big primes). The integer grows dynamically depending on available system memory.

2. Floating Point Numbers (float)

Floats represent real numbers with decimals, stored using IEEE 754 double precision (~15‚Äì16 significant digits).

PYTHON 
a = 10.25
print(a, type(a))           # 10.25 <class 'float'>

# Scientific notation
b = 1.5e3                   # 1500.0
c = 2.5e-4                  # 0.00025

# Float operations
pi = 3.14159
radius = 5.0
area = pi * radius ** 2
print(f"Area: {area}")      # Area: 78.53975
‚ö†Ô∏è Floating Point Precision Warning

Floating-point arithmetic can produce unexpected results due to binary representation:

print(0.1 + 0.2)  # 0.30000000000000004 (not exactly 0.3!)

# For precise decimal calculations, use the decimal module:
from decimal import Decimal
result = Decimal('0.1') + Decimal('0.2')
print(result)  # 0.3 (exact)

3. Complex Numbers (complex)

Complex numbers are represented as a + bj, where a is the real part and b is the imaginary part (denoted by j in Python).

PYTHON 
z = 3 + 5j
print(z)                    # (3+5j)
print(type(z))              # <class 'complex'>

# Accessing real and imaginary parts
print(z.real)               # 3.0
print(z.imag)               # 5.0

# Complex operations
z1 = 2 + 3j
z2 = 1 + 2j
print(z1 + z2)              # (3+5j)
print(z1 * z2)              # (-4+7j)
üí° Use Cases

Complex numbers are used in scientific computing, signal processing, electrical engineering, and quantum computing applications.

Arithmetic Operators

Python supports a comprehensive set of arithmetic operators for performing mathematical calculations:

Operator Description Example Output
+ Addition 5 + 3 8
- Subtraction 5 - 3 2
* Multiplication 4 * 3 12
/ Float division 5 / 2 2.5
// Floor division 5 // 2 2
% Modulus (remainder) 5 % 2 1
** Exponentiation 2 ** 3 8

Complete Examples

PYTHON 
a = 17
b = 5

print(f"Addition: {a} + {b} = {a + b}")           # 22
print(f"Subtraction: {a} - {b} = {a - b}")        # 12
print(f"Multiplication: {a} * {b} = {a * b}")     # 85
print(f"Division: {a} / {b} = {a / b}")           # 3.4
print(f"Floor Division: {a} // {b} = {a // b}")   # 3
print(f"Modulus: {a} % {b} = {a % b}")            # 2
print(f"Exponentiation: {a} ** 2 = {a ** 2}")     # 289
‚ö†Ô∏è Tricky Case: Floor Division with Negatives

Floor division // rounds down (toward negative infinity) for negative results:

print(5 // 2)    # 2
print(-5 // 2)   # -3 (rounds down, not toward zero!)
print(-5 / 2)    # -2.5

Type Conversion (Type Casting)

Python provides built-in functions to convert between different data types. This is essential when working with user input, file operations, or mixing different types in calculations.

From ‚Üí To Function Example Result
int ‚Üí float float() float(5) 5.0
float ‚Üí int int() int(9.8) 9
int/float ‚Üí complex complex() complex(3) (3+0j)
str ‚Üí int int() int("123") 123
int ‚Üí str str() str(50) "50"
str ‚Üí float float() float("3.14") 3.14

Practical Type Conversion Examples

PYTHON 
# Integer to Float
x = 10
y = float(x)
print(y, type(y))           # 10.0 <class 'float'>

# Float to Integer (truncates decimal)
pi = 3.14159
pi_int = int(pi)
print(pi_int)               # 3

# String to Integer
age_str = "25"
age = int(age_str)
print(age + 5)              # 30

# Number to String (for concatenation)
score = 95
message = "Your score is: " + str(score)
print(message)              # Your score is: 95

# Creating Complex Numbers
c = complex(3, 4)           # 3 + 4j
print(c)
‚ö†Ô∏è Type Conversion Errors

Invalid conversions will raise exceptions:

int("4a7")      # ‚ùå ValueError: invalid literal
int("3.14")     # ‚ùå ValueError: invalid literal
float("hello")  # ‚ùå ValueError: could not convert

# Correct approaches:
int(float("3.14"))  # ‚úÖ 3 (convert to float first, then to int)

Strings (str)

Strings are immutable sequences of Unicode characters. They can be wrapped in single quotes, double quotes, or triple quotes for multi-line strings.

PYTHON 
s = "Python"
print(s)                    # Python
print(type(s))              # <class 'str'>

# String indexing (0-based)
print(s[0])                 # P (first character)
print(s[-1])                # n (last character)

# String slicing
print(s[0:3])               # Pyt
print(s[2:])                # thon
print(s[:4])                # Pyth

String Immutability

Strings are immutable ‚Äî any modification creates a new string object:

PYTHON 
text = "Hello"
# text[0] = "h"  # ‚ùå TypeError: 'str' object does not support item assignment

# To modify, create a new string:
text = "h" + text[1:]
print(text)                 # hello

String Operations

PYTHON 
# Concatenation
first = "Hello"
last = "World"
full = first + " " + last
print(full)                 # Hello World

# Repetition
print("Python! " * 3)       # Python! Python! Python!

# Length
print(len("Python"))        # 6

# Membership
print("Py" in "Python")     # True
print("Java" in "Python")   # False

Useful String Methods

PYTHON 
text = "  Python Programming  "

print(text.upper())         # PYTHON PROGRAMMING
print(text.lower())         # python programming
print(text.strip())         # Python Programming (removes spaces)
print(text.replace("Python", "Java"))  # Java Programming
print(text.split())         # ['Python', 'Programming']
print("Hello".startswith("He"))  # True
print("World".endswith("ld"))    # True
Interview Tip

Python interns small strings (typically those containing ASCII letters only) for memory optimization and faster comparisons. This is why "hello" is "hello" returns True, but it's not guaranteed for all strings.

1. Lists (list)

Lists are ordered, mutable collections that allow duplicates. They're one of the most versatile data structures in Python.

PYTHON 
# Creating lists
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.5, True]
nested = [[1, 2], [3, 4], [5, 6]]

print(numbers)              # [1, 2, 3, 4, 5]
print(type(numbers))        # <class 'list'>

# Accessing elements
print(numbers[0])           # 1
print(numbers[-1])          # 5

# Modifying lists (mutable)
numbers[1] = 10
print(numbers)              # [1, 10, 3, 4, 5]

# Adding elements
numbers.append(6)           # Add to end
numbers.insert(0, 0)        # Insert at position
print(numbers)              # [0, 1, 10, 3, 4, 5, 6]

# Removing elements
numbers.remove(10)          # Remove specific value
popped = numbers.pop()      # Remove and return last item
print(numbers)              # [0, 1, 3, 4, 5]

2. Tuples (tuple)

Tuples are ordered, immutable collections. Once created, their elements cannot be modified.

PYTHON 
# Creating tuples
t = (1, 2, 3)
print(t, type(t))           # (1, 2, 3) <class 'tuple'>

# Accessing elements
print(t[0])                 # 1

# Tuples are immutable
# t[0] = 10  # ‚ùå TypeError: 'tuple' object does not support item assignment
‚ö†Ô∏è Important: Comma Creates Tuple, Not Parentheses! 
x = (5)      # int (just parentheses for grouping)
y = (5,)     # tuple (comma makes it a tuple)

print(type(x))  # <class 'int'>
print(type(y))  # <class 'tuple'>

3. Range (range)

Range represents an immutable sequence of numbers, commonly used in loops.

PYTHON 
# Creating ranges
r1 = range(5)               # 0, 1, 2, 3, 4
r2 = range(1, 6)            # 1, 2, 3, 4, 5
r3 = range(0, 10, 2)        # 0, 2, 4, 6, 8 (step of 2)

# Using in loops
for i in range(5):
    print(i, end=" ")       # 0 1 2 3 4

# Converting to list
print(list(range(5)))       # [0, 1, 2, 3, 4]

üéØ List vs Tuple Comparison

  • Lists: Mutable, use when you need to modify data
  • Tuples: Immutable, faster, use for fixed data or as dict keys
  • Syntax: Lists use [], tuples use ()
  • Use tuples for returning multiple values from functions

Dictionaries (dict)

Dictionaries are mutable collections of key-value pairs. They provide fast lookup times and are fundamental to Python programming.

PYTHON 
# Creating dictionaries
student = {
    "name": "Ajay",
    "age": 25,
    "course": "Python",
    "grade": "A"
}

print(student)              # {'name': 'Ajay', 'age': 25, ...}
print(type(student))        # <class 'dict'>

# Accessing values
print(student["name"])      # Ajay
print(student.get("age"))   # 25

# Adding/Modifying entries
student["email"] = "ajay@example.com"
student["age"] = 26

# Removing entries
del student["course"]
popped = student.pop("grade")

# Dictionary methods
print(student.keys())       # dict_keys(['name', 'age', 'email'])
print(student.values())     # dict_values(['Ajay', 26, 'ajay@...'])
print(student.items())      # dict_items([('name', 'Ajay'), ...])

Dictionary Iteration

PYTHON 
# Iterate over keys
for key in student:
    print(key)

# Iterate over key-value pairs
for key, value in student.items():
    print(f"{key}: {value}")
üí° Dictionary Requirements

Keys must be hashable (immutable types like str, int, tuple). Lists and dictionaries cannot be used as keys because they're mutable.

Interview Tip

Dictionaries use hash tables internally, providing average O(1) time complexity for lookups, insertions, and deletions. They form the foundation of JSON data and object serialization in Python.

Sets (set)

Sets are unordered, mutable collections of unique items. They're perfect for membership testing and removing duplicates.

PYTHON 
# Creating sets
numbers = {1, 2, 3, 2, 4, 3, 5}
print(numbers)              # {1, 2, 3, 4, 5} (duplicates removed)
print(type(numbers))        # <class 'set'>

# Creating empty set (use set(), not {})
empty = set()               # Correct
# empty = {}  # This creates an empty dict!

# Adding elements
numbers.add(6)
numbers.update([7, 8, 9])

# Removing elements
numbers.remove(9)           # Raises KeyError if not found
numbers.discard(10)         # Does not raise error if not found

Set Operations

PYTHON 
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Union (all elements from both)
print(a | b)                # {1, 2, 3, 4, 5, 6}
print(a.union(b))

# Intersection (common elements)
print(a & b)                # {3, 4}
print(a.intersection(b))

# Difference (in a but not in b)
print(a - b)                # {1, 2}
print(a.difference(b))

# Symmetric Difference (in either, but not both)
print(a ^ b)                # {1, 2, 5, 6}
‚úÖ Common Use Cases

Removing duplicates: Convert list to set and back
Membership testing: x in set is O(1) on average
Mathematical operations: Union, intersection, difference

Boolean (bool)

Boolean type represents two logical values: True and False. Booleans are actually subclasses of integers in Python.

PYTHON 
# Boolean values
is_active = True
is_valid = False

print(type(is_active))      # <class 'bool'>

# Booleans as integers
print(True + True)          # 2
print(True * 5)             # 5
print(False + 10)           # 10

# Boolean equivalence
print(True == 1)            # True
print(False == 0)           # True

Boolean Conversion

Any value can be evaluated in a boolean context using bool():

PYTHON 
# Truthy values
print(bool(1))              # True
print(bool("Python"))       # True
print(bool([1, 2, 3]))      # True

# Falsy values
print(bool(0))              # False
print(bool(""))             # False (empty string)
print(bool([]))             # False (empty list)
print(bool(None))           # False
üí° Falsy Values in Python

These values evaluate to False: False, None, 0, 0.0, '' (empty string), [] (empty list), {} (empty dict), set() (empty set)

NoneType (None)

None represents the absence of value or null equivalent in Python. It's a singleton object ‚Äî there's only one None in Python.

PYTHON 
x = None
print(x)                    # None
print(type(x))              # <class 'NoneType'>

# Checking for None (use 'is', not '==')
if x is None:
    print("x is None")      # Correct way

if x == None:
    print("x is None")      # Works but not recommended

Common Use Cases

PYTHON 
# Default function arguments
def greet(name=None):
    if name is None:
        name = "Guest"
    print(f"Hello, {name}!")

greet()                     # Hello, Guest!
greet("Alice")              # Hello, Alice!

# Placeholder for variables
result = None
# ... some processing ...
if result is not None:
    print(result)
üí° Why Use 'is' with None?

Use is None instead of == None because is checks object identity (faster), while == checks equality (can be overridden).

Mutability: A Critical Concept

Understanding mutability is crucial for avoiding bugs and writing efficient Python code.

Mutable Data Types

Can be modified after creation:

list dict set

Immutable Data Types

Cannot be modified after creation (modifications create new objects):

int float complex str tuple bool NoneType

Demonstration

PYTHON 
# Mutable: Changes affect original
x = [1, 2, 3]
y = x
y.append(4)
print(x)                    # [1, 2, 3, 4] (changed!)

# Immutable: Creates new object
a = "Hello"
b = a
b = b + " World"
print(a)                    # Hello (unchanged)
print(b)                    # Hello World
Interview Tip

This is a common interview question! Understanding mutability helps you predict behavior when passing arguments to functions and prevents unexpected modifications to data.
