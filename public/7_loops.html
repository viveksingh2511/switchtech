Introduction to Loops

What is a Loop and Why Do We Need It?

A loop is used to repeat a block of code multiple times without writing the same code again and again. Loops make code shorter, cleaner, and more efficient.

‚ùå Without Loop
print("Hello")
print("Hello")
print("Hello")
print("Hello")
print("Hello")

Repetitive and inefficient!

‚úÖ With Loop
for i in range(5):
    print("Hello")

Clean and efficient!

Types of Loops in Python

üîÅ For Loop

Purpose: Iterate over a sequence

Iterations: Fixed/Known

Best For: Lists, strings, ranges

üîÉ While Loop

Purpose: Repeat based on condition

Iterations: Unknown/Variable

Best For: User input, conditions

üí° Key Concept

Loops help us automate repetitive tasks, process collections of data, and implement algorithms efficiently. Understanding loops is fundamental to mastering Python programming.

‚öñÔ∏è

For Loop vs While Loop

Feature For Loop While Loop
Use Case When we know number of iterations When we don't know number of iterations
Condition Uses range() or iterables Uses a boolean condition
Best For Traversing collections Repeating until condition breaks
Termination Ends when sequence exhausted Ends when condition becomes False
Control Variable Automatically managed Manually updated
Risk Low (less chance of infinite loop) Higher (if condition never False)

‚úÖ When to Use Which?
  • Use For Loop: When iterating over sequences (lists, strings, ranges)
  • Use While Loop: When repeating based on conditions (user input, thresholds)

For Loop - Complete Guide

Fixed Iterations Sequence Traversal Collection Processing

1. Using range()

Concept: range(start, stop, step) generates a sequence of numbers. Used when we know how many times to loop.

PYTHON 
# Example 1: Basic range
for i in range(5):
    print(i)
# Output: 0 1 2 3 4

# Example 2: Range with start and stop
for i in range(2, 6):
    print(i)
# Output: 2 3 4 5

# Example 3: Range with step
for i in range(1, 10, 2):
    print(i)
# Output: 1 3 5 7 9

# Example 4: Create list from range
nums = list(range(4))
print(nums)
# Output: [0, 1, 2, 3]

# Example 5: Sum using range
total = sum(range(1, 6))
print(total)
# Output: 15 (1+2+3+4+5)
üîÑ range() Flow
range(1, 6)
‚Üì

2. Using in (Membership Iteration)

Concept: in lets us loop through iterable items directly without using an index.

PYTHON 
# Example 1: Iterate string
for ch in "Python":
    print(ch)
# Output: P y t h o n

# Example 2: Iterate list
colors = ["red", "blue", "green"]
for c in colors:
    print(c)
# Output: red blue green

# Example 3: Check membership
if "blue" in colors:
    print("Found!")
# Output: Found!

# Example 4: Square numbers
for num in [1, 2, 3, 4]:
    print(num**2)
# Output: 1 4 9 16

# Example 5: Iterate set
for word in {"apple", "mango"}:
    print(word)

3. Using enumerate()

Concept: enumerate() gives both index and value when iterating over a sequence.

PYTHON 
# Example 1: Basic enumerate
fruits = ["apple", "banana", "mango"]
for i, f in enumerate(fruits):
    print(i, f)
# Output:
# 0 apple
# 1 banana
# 2 mango

# Example 2: Start index from 1
for i, f in enumerate(fruits, start=1):
    print(i, f)
# Output:
# 1 apple
# 2 banana
# 3 mango

# Example 3: Enumerate string
for i, ch in enumerate("Python", start=1):
    print(f"{i}: {ch}")

# Example 4: Student marks
marks = [90, 85, 80]
for i, m in enumerate(marks):
    print(f"Student{i+1} ‚Üí {m}")

# Example 5: Convert to list of tuples
data = ["x", "y", "z"]
indexed = list(enumerate(data))
print(indexed)
# Output: [(0, 'x'), (1, 'y'), (2, 'z')]
Interview Tip

enumerate() vs range(): Use enumerate() when you need both index and value. Use range(len(list)) when you only need the index. enumerate() is more Pythonic!

4. Using zip()

Concept: zip() combines two or more iterables element by element.

PYTHON 
# Example 1: Zip two lists
names = ["Vanya", "Deepak", "Abhishek"]
marks = [90, 85, 88]
for n, m in zip(names, marks):
    print(f"{n} ‚Üí {m}")
# Output:
# Vanya ‚Üí 90
# Deepak ‚Üí 85
# Abhishek ‚Üí 88

# Example 2: Create tuples
x = [1, 2, 3]
y = [10, 20, 30]
print(list(zip(x, y)))
# Output: [(1,10), (2,20), (3,30)]

# Example 3: Zip string and list
for a, b in zip("abc", [100, 200, 300]):
    print(a, b)

# Example 4: Create dictionary
roll = [101, 102, 103]
names = ["Vanya", "Deepak", "Amit"]
print(dict(zip(roll, names)))
# Output: {101: 'Vanya', 102: 'Deepak', 103: 'Amit'}

# Example 5: Zip three lists
l1 = [1, 2]
l2 = [3, 4]
l3 = [5, 6]
print(list(zip(l1, l2, l3)))
# Output: [(1,3,5), (2,4,6)]

6. List Comprehension

Concept: A compact way to create new lists by applying expressions in a loop.

PYTHON 
# Example 1: Square numbers
numbers = [1, 2, 3, 4]
squares = [n**2 for n in numbers]
print(squares)
# Output: [1, 4, 9, 16]

# Example 2: Double values
doubled = [n*2 for n in numbers]
print(doubled)
# Output: [2, 4, 6, 8]

# Example 3: Filter evens
evens = [n for n in range(10) if n % 2 == 0]
print(evens)
# Output: [0, 2, 4, 6, 8]

# Example 4: Uppercase strings
words = ["hello", "world"]
upper = [w.upper() for w in words]
print(upper)
# Output: ['HELLO', 'WORLD']

# Example 5: Conditional expression
labels = ["even" if n%2==0 else "odd" for n in range(5)]
print(labels)
# Output: ['even', 'odd', 'even', 'odd', 'even']

While Loop - Complete Guide

Condition-Based Unknown Iterations User Input

What is a While Loop?

A while loop repeatedly executes a block of code as long as a given condition remains True. When the condition becomes False, the loop stops automatically. It's most useful when the number of iterations is not known in advance.

Syntax and Flow

SYNTAX 
while condition:
    # block of code
    # update condition variable
üîÑ While Loop Flow
Test Condition
‚Üì
If True ‚Üí Execute Body
‚Üì
Update Variable
‚Üì
Check Again
‚Üì
If False ‚Üí Exit Loop

While Loop Examples

Case 1: Countdown (Basic Control Loop)

Concept: Decrement a number each time until it reaches 0.

PYTHON 
count = 5
while count > 0:
    print(count)
    count -= 1
print("Blast off!")

# Output:
# 5
# 4
# 3
# 2
# 1
# Blast off!

Case 2: User Input Until Quit

Concept: Keep asking for input repeatedly until the user types a specific keyword.

PYTHON 
word = ""
while word != "quit":
    word = input("Enter a word (quit to exit): ")
    if word != "quit":
        print(f"You typed: {word}")
print("Goodbye!")

# Sample interaction:
# Enter a word (quit to exit): hello
# You typed: hello
# Enter a word (quit to exit): python
# You typed: python
# Enter a word (quit to exit): quit
# Goodbye!

Case 3: Sum Until Threshold

Concept: Continuously add numbers until a target value is reached.

PYTHON

i = 1
while total < 50:
    total += i
    print(f"Added {i}, total now: {total}")
    i += 1
print(f"Final total: {total}")

# Added 1, total now: 1
# Added 2, total now: 3
# Added 3, total now: 6

Case 4: Searching in a List

Concept: Use while loop to find an element.

PYTHON 
nums = [10, 20, 30, 40, 50]
target = 30
i = 0

while i < len(nums) and nums[i] != target:

# Output: Found 30 at index: 2

Advanced While Loop Patterns

1. Infinite Loop with break

PYTHON 
# while True creates endless loop
while True:
    word = input("Type 'stop' to exit: ")
    if word == "stop":
        break
    print(f"You typed: {word}")
print("Loop exited!")

2. Loop with continue

PYTHON 
# Skip even numbers
n = 0
while n < 10:
    n += 1
    if n % 2 == 0:
        continue
    print(n)

# Output: 1 3 5 7 9

3. Nested While Loop

PYTHON 
i = 1
while i <= 3:
    j = 1
    while j <= 2:
        print(f"i={i}, j={j}")
        j += 1
    i += 1

# Output:
# i=1, j=1
# i=1, j=2
# i=2, j=1
# i=2, j=2
# i=3, j=1
# i=3, j=2

4. Data Validation

PYTHON 
# Keep asking until valid input
num = -1
while num < 0:
    num = int(input("Enter positive number: "))
    if num < 0:
        print("‚ùå Must be positive!")
print(f"‚úì You entered: {num}")

5. Loop with else

PYTHON 
# else executes when condition becomes False
n = 3
while n > 0:
    print(n)
    n -= 1
else:
    print("Loop ended normally")

# Output:
# 3
# 2
# 1
# Loop ended normally
‚ö†Ô∏è Common Mistakes
  • Infinite loops: Forgetting to update the variable
  • Wrong condition: Condition never becomes False
  • Off-by-one: Incorrect initialization or update
  • Misplaced break: Breaking at wrong point

üéØ While Loop Best Practices

  • Always ensure condition will eventually become False
  • Use break carefully to exit loops early
  • Prefer For Loops if iterations are known
  • Use While Loops for user input, conditions, or indefinite processes
  • Test for infinite loops before deployment

When to Use While Loop

‚úÖ Use While When

‚Ä¢ Continuous user input

‚Ä¢ Unknown iteration count

‚Ä¢ Condition-based repetition

‚Ä¢ Menu-driven programs

‚Ä¢ Data validation loops

‚Ä¢ Background monitoring

‚ùå Avoid While When

‚Ä¢ Iterating over collections

‚Ä¢ Known number of iterations

‚Ä¢ Processing sequences

‚Ä¢ Simple counting loops

‚Ä¢ Use For Loop instead!

Real-World Loop Examples

Example 1: Password Validator

PYTHON 
max_attempts = 3
attempts = 0

while attempts < max_attempts:
    password = input("Enter password: ")
    
    # Check password criteria
    if len(password) < 8:
        print("‚ùå Too short (min 8 characters)")
    elif not any(c.isdigit() for c in password):
        print("‚ùå Must contain a number")
    elif not any(c.isupper() for c in password):
        print("‚ùå Must contain uppercase")
    else:
        print("‚úì Password accepted!")
        break
    
    attempts += 1
    remaining = max_attempts - attempts
    if remaining > 0:
        print(f"Attempts remaining: {remaining}\n")
else:
    print("‚ùå Maximum attempts reached!")

Example 5: Prime Number Finder

PYTHON 
def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

# Find all primes up to 50
primes = []
for num in range(2, 51):
    if is_prime(num):
        primes.append(num)

print("Prime numbers up to 50:")
print(primes)

# Using list comprehension
primes_compact = [n for n in range(2, 51) if is_prime(n)]
print("\nUsing comprehension:", primes_compact)

Complete Summary

üéØ For Loop Essentials

  • range() ‚Üí Generate number sequences (start, stop, step)
  • in ‚Üí Direct iteration over sequences
  • enumerate() ‚Üí Get both index and value
  • zip() ‚Üí Iterate multiple sequences together
  • Collections: Lists, tuples, dicts, sets
  • Comprehensions: One-line list creation
  • Use when: Number of iterations is known

üéØ While Loop Essentials

  • Condition-based: Repeats while condition is True
  • Manual control: Must update loop variable
  • Flexible: Unknown number of iterations
  • break: Exit loop early
  • continue: Skip current iteration
  • else: Executes after normal completion
  • Use when: Iterations depend on conditions

Quick Comparison

Aspect For Loop While Loop
Syntax for item in sequence: while condition:
Best For Sequences, collections Conditions, user input
Iterations Fixed/Known Unknown/Variable
Control Automatic Manual update needed
Risk Low (finite) Higher (can be infinite)
Common Use Data processing Interactive programs
Final Interview Tips
  • Loop Choice: Always prefer for loop when possible (more Pythonic)
  • Enumerate: Use instead of range(len(list)) for cleaner code
  • Zip: Perfect for parallel iteration of multiple sequences
  • Comprehensions: More efficient than traditional loops
  • While risks: Always ensure termination condition
  • Break/Continue: Know when and how to control loop flow
