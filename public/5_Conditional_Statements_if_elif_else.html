Introduction to Conditional Statements

In programming, conditionals determine which part of the code should be executed based on whether a condition (expression) evaluates to True or False.

Decision Making Control Flow Boolean Logic

Purpose of Conditionals

Conditionals add decision-making ability to your programs, allowing them to respond differently based on various conditions and user inputs.

Key Components

  • if: Executes a block if the condition is True
  • elif: Tests another condition if the previous ones are False
  • else: Executes only when all previous conditions are False

Boolean Evaluation

Any expression that results in True or False, such as comparisons (x > 10, name == "Alice", etc.).

PYTHON 
temperature = 30
if temperature > 25:
    print("It's warm outside.")
# Output: It's warm outside.
üí° How It Works

The message prints only when the condition temperature > 25 evaluates to True. This is the foundation of all decision-making in programming!

The if Statement

The if statement is the simplest form of decision-making. It allows your program to execute a block of code only when a specific condition is met.

Syntax Structure
if condition:
    statement(s)

Key Characteristics

  • Begins with the keyword if
  • Followed by a condition (expression that evaluates to True/False)
  • Ends with a colon (:)
  • The code block underneath must be indented (typically 4 spaces)

Basic Example

PYTHON 
x = 10
if x > 5:
    print("x is greater than 5")
# Output: x is greater than 5

How It Works (Step-by-Step)

  1. Python evaluates the condition x > 5
  2. Since 10 > 5 is True, the indented statement executes
  3. The message is printed to the console
  4. If the condition were False, Python would skip the indented block entirely

Multiple Statements in if Block

PYTHON 
age = 20
if age >= 18:
    print("You are an adult")
    print("You can vote")
    print("You have full legal rights")
# All three statements execute because age >= 18 is True

# ‚úÖ Correct

The else Statement

The else statement handles the "alternative" branch‚Äîwhat to do when the if condition is False. It provides a default action when the primary condition isn't met.

Syntax Structure
if condition:
    # executes when True
else:
    # executes when False

Basic Example

PYTHON 
x = 2
if x > 5:
    print("x is greater than 5")
else:
    print("x is 5 or smaller")
# Output: x is 5 or smaller

How It Works

  • Python first evaluates the if condition
  • If True, only the if block executes
  • If False, Python automatically jumps to the else block
  • Only one branch ever executes‚Äînever both

The elif Statement (else if)

elif allows multiple branching decisions‚Äîchecking more than one condition in sequence. It's short for "else if" and is used when you have more than two possible outcomes.

Syntax Structure
if condition1:
    statement1
elif condition2:
    statement2
elif condition3:
    statement3
else:
    fallback_statement

Basic Example

PYTHON 
x = 10

if x < 0:
    print("Negative")
elif x == 0:
    print("Zero")
elif x < 10:
    print("Single digit positive")
else:
    print("Two-digit or larger")
# Output: Two-digit or larger

How It Works (Execution Flow)

  1. Python checks conditions in order from top to bottom
  2. Once a condition is True, it executes that block
  3. Skips all remaining elif and else blocks
  4. If no condition is True, the else block executes (if present)

Nested if-else Statements

Nested conditional statements place one if-else inside another for more complex, multi-level decision-making.

When to Use Nested Conditionals

  • When you need to check a condition only if another condition is already true
  • For hierarchical or dependent decision-making
  • When different criteria apply at different levels

Basic Example: Age Category

PYTHON 
age = 25

if age >= 18:
    print("You are an adult")
    if age >= 60:
        print("Category: Senior Citizen")
    else:
        print("Category: Working Age Adult")
else:
    print("You are a minor")
    if age >= 13:
        print("Category: Teenager")
    else:
        print("Category: Child")

# Output: You are an adult
#         Category: Working Age Adult

Step-by-Step Execution

  1. Check outer condition: age >= 18 ‚Üí True
  2. Enter the outer if block
  3. Check inner condition: age >= 60 ‚Üí False
  4. Execute inner else block

Shorthand (Ternary) if-else

A compact form of if-else used for simple one-line conditions. Also known as the ternary operator or conditional expression.

Syntax Structure
value_if_true if condition else value_if_false

Basic Examples

PYTHON 
# Simple even/odd check
x = 5
result = "Even" if x % 2 == 0 else "Odd"
print(result)  # Output: Odd

# Comparing two numbers
a, b = 10, 20
max_value = a if a > b else b
print(f"Maximum: {max_value}")  # Output: Maximum: 20

# Age category
age = 25
category = "Adult" if age >= 18 else "Minor"
print(category)  # Output: Adult

How It Works

  • If the condition is True, the first expression (value_if_true) is returned
  • If the condition is False, the second expression (value_if_false) is returned
  • Can be assigned to variables or used directly in expressions

Comparison: Traditional vs Ternary

PYTHON 
# Traditional if-else (5 lines)
score = 75
if score >= 50:
    status = "Pass"
else:
    status = "Fail"

# Ternary operator (1 line)
score = 75
status = "Pass" if score >= 50 else "Fail"

# Both produce the same result: Pass

Practical Examples

PYTHON 
# Discount eligibility
purchase = 150
discount = 20 if purchase > 100 else 10
print(f"Discount: {discount}%")  # Output: Discount: 20%

# Shipping fee
order_total = 80
shipping = 0 if order_total >= 100 else 5
print(f"Shipping: ${shipping}")  # Output: Shipping: $5

# Access level
is_admin = True
access_level = "Full Access" if is_admin else "Limited Access"
print(access_level)  # Output: Full Access

# Temperature message
temp = 32
message = "It's hot!" if temp > 30 else "It's comfortable"
print(message)  # Output: It's hot!

Using Ternary in Print Statements

# Output: Odd

Nested Ternary (Use with Caution)

PYTHON

‚ö†Ô∏è When to Avoid Ternary

Don't use ternary operators for:

  • Complex conditions with multiple statements
  • Deeply nested ternary expressions (hard to read)
  • When code clarity is more important than brevity
‚úÖ Best Use Cases

Perfect for: Simple variable assignments, quick value selections, default parameters, and one-line status checks where readability isn't compromised.

Logical Operators in Conditions

Logical operators combine multiple conditions into one expression, enabling more complex decision-making.

and

ALL conditions must be True

or

AT LEAST ONE must be True

not

Inverts the condition (True ‚Üî False)

The and Operator

Returns True only if all conditions are True.

    print("Eligible to vote")
# Output: Eligible to vote

# Login validation
username = "admin"
password = "secret"
if username == "admin" and password == "secret":
    print("Login successful")

# Range checking
score = 75
if score >= 0 and score <= 100:
    print("Valid score")

The not Operator

Inverts a Boolean value: True becomes False, and vice versa.

PYTHON

    print("You can go outside!")
# Output: You can go outside!

# Checking if NOT logged in

if not logged_in:
    print("Please log in to continue")

# Inverting conditions
is_member = True
if not is_member:
    print("Sign up for membership")
else:
    print("Welcome back, member!")

    print("Can rent a car")

    print("Extra discount applied!")

    print("Can moderate content")
Interview Tip

Logical operators follow short-circuit evaluation: For and, if the first condition is False, Python doesn't check the rest. For or, if the first condition is True, Python stops checking. This improves efficiency!

Key Takeaways

üéØ Core Concepts

  • if: Single condition check - executes block only if True
  • else: Alternative action when if condition is False
  • elif: Multiple branching checks - tests conditions in sequence
  • Nested if: Step-by-step hierarchical decisions
  • Ternary expression: Compact single-line if-else for simple cases
  • Comparison operators: ==, !=, >, <, >=, <= for comparisons
  • Logical operators: and, or, not for combining conditions
  • Indentation: Critical for defining code blocks in Python

Decision-Making Best Practices

  • Use elif instead of multiple if statements when conditions are mutually exclusive
  • Avoid deeply nested conditionals (max 3-4 levels)
  • Place most likely conditions first for better performance
  • Use ternary operators only for simple, readable cases
  • Always consider the else case, even if it's just error handling
  • Keep conditions simple and readable‚Äîbreak complex logic into smaller functions

Common Pitfalls to Avoid

‚ö†Ô∏è Watch Out For
  • Using = instead of == in conditions
  • Forgetting the colon : after if/elif/else statements
  • Incorrect indentation causing unexpected behavior
  • Testing for equality with floats (use ranges instead)
  • Not handling all possible cases (always include else for safety)
