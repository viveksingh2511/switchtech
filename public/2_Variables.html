Introduction to Variables

In Python, a variable is simply a name for a value stored in memory. Think of a variable as a labeled container: you store a value in it, and you use its label to retrieve or update the value later.

Python variables work with numbers, text, or other data types ‚Äî and Python automatically figures out the type from your assignment, a feature called dynamic typing.

Variable Concept Visualization

x ‚Üí 10

Variable "x" points to value "10" in memory

Dynamic Typing in Action

PYTHON 
x = 10         # Integer
print(x, type(x))  # Output: 10 <class 'int'>

x = "Hello"    # Now x is a string
print(x, type(x))  # Output: Hello <class 'str'>
üí° Key Concept: Dynamic Typing

No declaration is needed in advance ‚Äî assign a value, and Python takes care of the rest. The variable's type can change during program execution, giving Python tremendous flexibility.

‚úÖ Benefits of Variables

Reusability: Store a value once, use it many times throughout your program
Readability: Use descriptive names to make code self-documenting
Flexibility: Update values without rewriting code

Rules for Variable Names

Proper naming ensures that your code is valid and readable. Python enforces strict rules for variable names:

  • Can include: Letters (A‚ÄìZ, a‚Äìz), digits (0‚Äì9), and underscores (_)
  • Must not start with a digit ‚Äî 2name is invalid
  • No spaces allowed ‚Äî use underscores instead
  • No special characters ‚Äî avoid @, #, $, %, etc.
  • Case-sensitive ‚Äî age, Age, and AGE are three different variables
  • Cannot use Python keywords ‚Äî like if, for, class, while

Valid vs Invalid Variable Names

‚úÖ Valid Names

score = 90
student_name = "Amy"
_private = 100
name2 = "Bob"
userName = "Alice"

‚ùå Invalid Names

2score = 5      # Starts with digit
my-score = 6    # Contains hyphen
my score = 7    # Contains space
for = 10        # Python keyword
my@name = "X"   # Special character
‚ö†Ô∏è Case Sensitivity Warning

Remember that Python is case-sensitive! The variables username, userName, UserName, and USERNAME are all different variables. Be consistent with your naming style throughout your code!

Naming Conventions (PEP 8)

Python's style guide, PEP 8, recommends specific conventions to make code consistent and professional across the Python community.

snake_case CONSTANTS _internal

1. Use snake_case for Regular Variables

Lowercase letters with underscores separating words:

PYTHON 
student_name = "John"
total_score = 95
user_age = 25
max_attempts = 3

2. Use UPPERCASE for Constants

Values that should not change during program execution:

PYTHON 
PI = 3.14159
MAX_USERS = 100
DEFAULT_TIMEOUT = 30
DATABASE_URL = "localhost:5432"

3. Special Naming Patterns

  • Single leading underscore (_variable): Signals internal use or "private" variable
  • Double leading underscore (__variable): Special use inside classes for name-mangling
PYTHON 
_internal_var = 10     # Weak "internal use" indicator
__name_mangle = 20     # Strong private indicator (classes)

4. Choose Descriptive Names

Good variable names make code self-documenting:

‚úÖ Descriptive Names

max_score = 100
student_count = 25
is_active = True
user_email = "user@example.com"

‚ùå Unclear Names

ms = 100         # What is ms?
sc = 25          # Unclear
x = True         # Not descriptive
a = "user@..."   # Too generic

üéØ PEP 8 Best Practices

  • Use descriptive names that clearly indicate purpose
  • Follow snake_case for regular variables
  • Use UPPERCASE for constants
  • Avoid single-letter names (except for counters in loops)
  • Be consistent throughout your codebase

Assigning Values to Variables

Assign a value to a variable using the = operator. The variable is created at the moment of assignment ‚Äî no prior declaration needed!

Basic Assignment

PYTHON 
name = "John"
age = 25
height = 5.8
is_student = True

print(name)     # Output: John
print(age)      # Output: 25
print(height)   # Output: 5.8

Reassigning Variables

You can reassign values (and even change types) at any time:

PYTHON 
age = 25
print(age, type(age))  # Output: 25 <class 'int'>

age = "Twenty-five"
print(age, type(age))  # Output: Twenty-five <class 'str'>

Copying Variables

Assign one variable's value to another:

PYTHON 
original = 100
copy = original
print(copy)     # Output: 100

original = 200
print(original) # Output: 200
print(copy)     # Output: 100 (unchanged)
Example: Creating and Printing Variables
PYTHON 
# Creating variables
score = 85
print(score)                # 85

# Changing value and type
score = "Eighty-five"
print(score)                # Eighty-five

# Simple mathematical operations
a = 7
b = 3
s = a + b
print(s)                    # 10
üí° Understanding Assignment

When you write x = 10, Python creates an object with value 10 in memory and makes the variable name 'x' point to it. Reassigning x = 20 makes 'x' point to a new object with value 20.

Multiple Assignments

Python provides elegant shortcuts for assigning values to multiple variables simultaneously, making your code more concise and readable.

1. Assign the Same Value to Multiple Variables

PYTHON 
x = y = z = 100
print(x, y, z)   # Output: 100 100 100

# All three variables reference the same value
print(x)  # 100
print(y)  # 100
print(z)  # 100

2. Assign Different Values in One Line

PYTHON 
a, b, c = 1, 2, 3
print(a, b, c)   # Output: 1 2 3

# Assigning different types
name, age, height = "Alice", 25, 5.6
print(name)      # Alice
print(age)       # 25
print(height)    # 5.6

3. Variable Swapping

Python makes swapping variables incredibly easy:

PYTHON 
x, y = 10, 20
print(f"Before swap: x = {x}, y = {y}")
# Output: Before swap: x = 10, y = 20

x, y = y, x  # Swap values
print(f"After swap: x = {x}, y = {y}")
# Output: After swap: x = 20, y = 10

Practical Examples

PYTHON 
# Multiple assignment examples
m = n = 5
print(m, n)                  # 5 5

# Assigning multiple values
x, y = 10, 20
print(x, y)                  # 10 20

# Mixed types in one assignment
name, score, passed = "Bob", 85, True
print(name, score, passed)   # Bob 85 True
‚ùå ValueError: Mismatched Assignment

Make sure the number of variables matches the number of values, or Python will raise a ValueError:

x, y = 10, 20, 30  # ‚ùå Error: too many values
x, y, z = 10, 20   # ‚ùå Error: not enough values

# Correct usage:
x, y, z = 10, 20, 30  # ‚úÖ Correct

Variable Scope: Local vs Global

Understanding scope is crucial for writing correct Python programs. Scope determines where in your code a variable can be accessed.

Global Variables Local Variables

Global Variables

Global variables are created outside any function and can be accessed anywhere in your program.

Local Variables

Local variables are created inside a function and exist only within that function.

Example: Understanding Scope
PYTHON 
x = 50               # Global variable

def demo():
    y = 10           # Local variable
    print(y)         # Can access y inside function
    print(x)         # Can access global variable x

demo()               # Prints: 10, then 50
print(x)             # Prints: 50 (global variable)
# print(y)           # ‚ùå Error: y is not defined globally!

Variable Scope Visualization

Global Scope
x = 50 ‚Üê Accessible everywhere
Local Scope (inside demo)
y = 10 ‚Üê Only accessible here

Accessing Global Variables Inside Functions

PYTHON 
count = 0           # Global variable

def increment():
    global count    # Declare we're using global count
    count += 1
    print(f"Count inside function: {count}")

increment()         # Count inside function: 1
print(f"Count outside function: {count}")  # Count outside function: 1

Complete Scope Example

PYTHON 
# Scope demonstration
message = "Global message"  # Global variable

def greet():
    greeting = "Hi"         # Local variable
    print(greeting)         # Hi (local variable)
    print(message)          # Global message (can access global)

greet()
print(message)              # Global message (works)
# print(greeting)             # ‚ùå NameError: greeting not defined
‚ùå Common Mistake: NameError

Use of local variables outside their scope leads to a NameError. Always ensure you're trying to access variables within their valid scope!

‚ö†Ô∏è Modifying Global Variables

To modify a global variable inside a function, you must use the global keyword. Without it, Python creates a new local variable with the same name, which can lead to confusing bugs!

Variable Type Detection and Conversion

Python provides built-in functions to check and convert variable types, essential for writing flexible and error-free code.

Checking Variable Types with type()

PYTHON 
x = 10
print(type(x))      # Output: <class 'int'>

x = "Hello"
print(type(x))      # Output: <class 'str'>

x = 3.14
print(type(x))      # Output: <class 'float'>

x = True
print(type(x))      # Output: <class 'bool'>

Type Conversion (Type Casting)

Convert variables from one type to another using built-in functions:

PYTHON 
# String to Integer
age_str = "25"
age_int = int(age_str)
print(age_int + 5)  # Output: 30

# Integer to String (for concatenation)
age = 25
message = "I am " + str(age) + " years old"
print(message)      # Output: I am 25 years old

# String to Float
price = "19.99"
price_float = float(price)
print(price_float)  # Output: 19.99

# Float to Integer (truncates decimal)
pi = 3.14159
pi_int = int(pi)
print(pi_int)       # Output: 3

Practical Type Conversion Examples

PYTHON 
# Example 1: User input (always returns string)
name = input("Enter your name: ")
age_input = input("Enter your age: ")
age = int(age_input)  # Convert string to int

print(f"{name} is {age} years old")
print(f"Next year, {name} will be {age + 1}")

# Example 2: Calculations with type conversion
num1 = "10"
num2 = "20"
result = int(num1) + int(num2)
print(result)  # Output: 30
‚ö†Ô∏è Type Conversion Errors

Type conversion can fail if the value cannot be converted. For example:

x = "hello"
y = int(x)  # ‚ùå ValueError: invalid literal for int()

Always ensure the string contains a valid number before converting!

Best Practices & Common Errors

‚úÖ Best Practices

  • Always use descriptive variable names (avoid single letters except as counters)
  • Follow PEP 8 naming conventions consistently
  • Initialize variables before using them
  • Be mindful of variable scope
  • Use type conversion when mixing different data types
  • Comment your code to explain complex variable usage

Common Mistakes to Avoid

‚ùå Error 1: Accessing Local Variables Outside Scope 
def calculate():
    result = 100

calculate()
print(result)  # ‚ùå NameError: name 'result' is not defined
‚ùå Error 2: Mixing Types Without Conversion 
age = 25
print("I am " + age + " years old")  # ‚ùå TypeError

# Fix:
print("I am " + str(age) + " years old")  # ‚úÖ
‚ùå Error 3: Using Reserved Keywords 
for = 10       # ‚ùå SyntaxError: invalid syntax
class = "A"    # ‚ùå SyntaxError: invalid syntax

# Fix: Use different names
loop_count = 10  # ‚úÖ
grade = "A"      # ‚úÖ
‚ùå Error 4: Mismatched Multiple Assignment 
x, y = 10, 20, 30  # ‚ùå ValueError: too many values

# Fix:
x, y, z = 10, 20, 30  # ‚úÖ
